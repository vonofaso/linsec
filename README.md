# LinSec — Linux Security Auditor

**LinSec** — консольный инструмент для локального аудита безопасности Linux-систем. Программа проверяет настройки операционной системы, сопоставляет результаты с рекомендациями **CIS Benchmarks** и **ФСТЭК России**, после чего формирует отчеты в форматах JSON и HTML.

## Возможности

Инструмент выполняет аудит по следующим направлениям:

- конфигурация SSH-сервера;
- учетные записи и парольная политика;
- права доступа к системным файлам;
- небезопасные и неиспользуемые службы;
- SUID- и SGID-файлы;
- открытые TCP-порты;
- параметры ядра и сетевого стека;
- настройки межсетевого экрана;
- параметры загрузки ядра;
- соответствие рекомендациям CIS ФСТЭК.

## Требования

- Linux;
- Python 3;
- права суперпользователя для полного аудита;

Инструмент ориентирован преимущественно на Debian-совместимые дистрибутивы, включая Ubuntu.

## Запуск

Перейдите в корневую директорию проекта:

```bash
cd linsec
```

Запустите аудит с правами суперпользователя:

```bash
sudo python3 main.py
```

После завершения аудита отчеты будут сохранены в директории `reports`.

## Модули аудита

### SSH

Модуль анализирует файл `/etc/ssh/sshd_config`.

Проверяются:

- запрет входа пользователя `root`;
- запрет пустых паролей;
- ограничение количества попыток аутентификации;
- отключение host-based-аутентификации;
- запрет пользовательских переменных окружения;
- параметры завершения неактивных сессий;
- использование login banner;
- настройка `IgnoreRhosts`.

### Пользователи

Проверяются:

- наличие UID `0` только у пользователя `root`;
- учетные записи с пустыми паролями;
- заблокированные учетные записи;
- автоматическое завершение неактивных сессий через `TMOUT`;
- блокировка после неудачных попыток входа;
- минимальная длина пароля;
- ограничение команды `su`;
- сроки действия паролей;
- уникальность UID;
- наличие `NOPASSWD` в конфигурации `sudo`;
- права домашних директорий;
- права доступа к пользовательским конфигурационным файлам.

### Файлы и службы

Модуль проверяет права и владельцев следующих объектов:

- `/etc/shadow`;
- `/etc/passwd`;
- `/etc/group`;
- `/etc/gshadow`;
- `/etc/crontab`;
- каталогов `cron.hourly`, `cron.daily`, `cron.weekly` и `cron.monthly`;
- файлов `sudoers`;
- стартовых скриптов и unit-файлов `systemd`;
- пользовательских cron-файлов;
- файлов в `/bin`, `/usr/bin`, `/lib` и `/lib64`.

Дополнительно проверяются:

- sticky bit на `/tmp`;
- пароль загрузчика GRUB;
- блокировка модуля `usb-storage`;
- состояние AppArmor;
- world-writable-файлы;
- использование теневых паролей;
- состояние CUPS;
- наличие и активность Telnet, FTP, Samba, NFS, rsync, NIS и rsh.

### SUID и SGID

Выполняется поиск файлов с установленными битами SUID или SGID.

Отдельно выявляются потенциально опасные SUID-файлы, включая:

- командные оболочки;
- интерпретаторы;
- текстовые редакторы;
- сетевые утилиты;
- средства отладки;
- контейнерные инструменты;
- команды управления файлами и правами доступа.

### Сеть и ядро

Проверяются:

- слушающие TCP-порты;
- потенциально опасные сетевые службы;
- состояние UFW или iptables;
- состояние Bluetooth;
- параметры IPv4 и IPv6;
- фильтрация ICMP;
- SYN Cookies;
- Reverse Path Filtering;
- ASLR;
- ограничения BPF, `ptrace`, `dmesg`, `kexec` и user namespaces;
- защита символьных и жестких ссылок;
- параметры загрузки ядра, включая IOMMU, TSX, `slab_nomerge` и `init_on_alloc`.

## Нормативная база

При разработке использовались:

- **CIS Ubuntu Linux 24.04 LTS Benchmark v1.0.0**;
- методические рекомендации ФСТЭК России по безопасной настройке Linux-систем.

Соответствующие документы находятся в директории `docs`.
